meta/

	This functionality is highly experimental and it's implementation is unfinished yet.

Metaconfiguration sample for DAO example:

    <?xml version="1.0"?>
    <!DOCTYPE metaconfiguration SYSTEM "meta.dtd">

    <metaconfiguration><classes>

        <class name="Employer" type="final">
            <properties>

                <property name="id" type="BigInteger" required="true" identifier="true" />
                <property name="name" type="String" size="255" required="true" />

                <property name="birthDate" type="Timestamp" required="true" />
                <property name="created"   type="Timestamp" required="true" />
                <property name="modified"  type="Timestamp" required="true" />

            </properties>

        <pattern name="DictionaryClass" />
        </class>

    </classes></metaconfiguration>

You can generate business and DAO classes, DB schema and various common utils from this configuration file automatically.
